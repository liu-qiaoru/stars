# Domain docs

本仓库采用 single-context 领域文档布局。server、worker、web 和 shared 共享“本地多模态媒体检索与编辑”领域语义。

## 开始领域工作前

- 如果根目录存在 `CONTEXT.md`，先读取它。
- 如果 `docs/adr/` 存在，读取与当前工作相关的 ADR。
- 文件尚不存在时继续工作，不把缺失本身视为阻塞；领域建模工作按需创建它们。

## 目录约定

```text
/
├── CONTEXT.md
└── docs/adr/
```

## 消费规则

- issue、规格、测试名称和代码设计使用 `CONTEXT.md` 中定义的领域词汇，避免无意引入同义词。
- 需要的新概念若未在词汇表中出现，应先判断它是术语漂移还是真实领域缺口。
- 方案若与已有 ADR 冲突，必须明确指出冲突和重新讨论的原因，不得静默覆盖。
