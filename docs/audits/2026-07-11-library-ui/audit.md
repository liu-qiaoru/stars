# 素材库 UI 审查

审查时间：2026-07-11

## 范围

素材库页面的默认折叠状态与文件展开状态，桌面视口 1280 × 720。

1. 默认折叠：[01-collapsed.png](./01-collapsed.png)
2. 文件展开：[02-expanded.png](./02-expanded.png)

## 结论

当前实现不应作为最终 UI。页面把“添加素材库”做成常驻的大卡片，把主要的素材浏览区域降成了右侧附属区域；素材库卡片内部又同时堆叠路径、三个统计块和两个同等级按钮，信息层级混乱。展开后文件列表继续嵌套卡片与边框，纵向密度低，长文件名的可读空间不足。

## 主要问题

1. 添加动作与高频浏览任务争夺首屏空间，主次关系反了。
2. 素材库卡片存在 panel、card、metric、file row 多层边框，视觉噪声过大。
3. 三个统计块像三个可点击卡片，但实际上只是只读数据。
4. “查看文件”和“扫描”视觉权重相同，没有明确主操作。
5. 展开区域缺少表头或稳定列结构，文件名、类型、状态扫读效率低。
6. 路径、数量和状态没有形成一条紧凑的信息摘要，导致单个素材库占用过高。
7. 截图只能确认视觉层级和布局问题；键盘焦点、缩放和读屏体验需要另行交互验证。

## 推荐方向

将页面改成单列素材库管理页：顶部只保留标题、总览和“添加素材库”按钮，添加表单放入按需展开区；每个素材库使用一条紧凑的可展开行，名称与路径在左，数量摘要居中，扫描状态与操作在右。展开后使用无外层卡片嵌套的文件表格/列表，并让文件名占据主要宽度。
